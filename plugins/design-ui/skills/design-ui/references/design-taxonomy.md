## Historical and stylistic references

- `Bauhaus`
- `Mondrianism`
- `suprematism`
- `dada graphic design`

## Experimental or opt-in only
Only use when the product, brand, or brief clearly supports it:
- `glitch`
- `ruined effect`
- `futuristic neon`
- `skeuomorphism`

## Product-context direction matrix
### Calm enterprise clarity
Use for: B2B SaaS, dashboards, trust-first products, operations-heavy tools
- Primary: `Swiss`, `modular layouts`
- Supporting: `muted colors`, `subtle colored shadows`, `lineart`
- Avoid: `glitch`, `futuristic neon`, decorative gradients with no structural role

### Expressive editorial technology
Use for: portfolios, thought leadership products, creative studios, brand-forward marketing pages
- Primary: `asymmetry`, `broken grid layouts`
- Supporting: bold typography, `authentic photography`, selective `duotone`
- Avoid: rigid dashboard repetition, default enterprise iconography, generic centered heroes

### Airy modern startup
Use for: startup landing pages, onboarding flows, product marketing, feature storytelling
- Primary: `floating`, `dynamic gradients`
- Supporting: `abstract geometrical`, `tailored illustrations`, soft depth
- Avoid: over-dense enterprise structure, too many accents, random style mixing

### Structured technical explainer
Use for: developer tools, docs, educational products, workflow explainers
- Primary: `modular layouts`, `geometric`
- Supporting: `lineart`, `isometric`, `material colors`
- Avoid: ornamental textures, expressive art references that reduce clarity, decorative motion with no information value

## Usage rules
- Choose exactly 1 primary design language.
- Add at most 1-2 supporting treatments.
- Add 1 imagery mode.
- Name an explicit avoid list.
- Do not mix contradictory styles without reason.
