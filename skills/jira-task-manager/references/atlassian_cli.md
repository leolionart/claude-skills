# Atlassian CLI

CLI wrappers for Jira and Confluence generated from the local MCP server definition in `.mcp.json`.

## Files

- `scripts/generate_atlassian_clis.sh` — imports `mcp-atlassian` from `.mcp.json`, regenerates local `config/mcporter.json`, compiles the shared binary, and refreshes both wrappers
- `scripts/mcp-atlassian` — compiled binary generated by `mcporter` + `bun`
- `scripts/jira-mcp` — Jira wrapper with lean defaults and `--output json` compatibility
- `scripts/confluence-mcp` — Confluence wrapper with `--output json` compatibility
- `.mcp.json` — local MCP definition with environment-variable placeholders for Jira/Confluence URLs and tokens

## Generate / Refresh

```bash
./scripts/generate_atlassian_clis.sh
```

What it does:
1. Import `mcp-atlassian` from local `.mcp.json` into `config/mcporter.json`
2. Validate the imported server with `mcporter list`
3. Compile `scripts/mcp-atlassian`
4. Recreate `scripts/jira-mcp` and `scripts/confluence-mcp`

## Common usage

### Jira

```bash
# Search issues
scripts/jira-mcp jira-search \
  --jql "project = AI AND status != Done" \
  --limit 10 \
  --output json

# Get one issue
scripts/jira-mcp jira-get-issue \
  --issue-key "AI-123" \
  --fields "summary,status,assignee,priority,issuetype,updated" \
  --output json

# List active sprints
scripts/jira-mcp jira-get-sprints-from-board \
  --board-id "168" \
  --state "active" \
  --limit 5 \
  --output json
```

### Confluence

```bash
# Search pages
scripts/confluence-mcp confluence-search \
  --query 'type=page AND space=BD' \
  --limit 5 \
  --output json

# Get page by ID
scripts/confluence-mcp confluence-get-page \
  --page-id '211539684' \
  --output json

# Update page
scripts/confluence-mcp confluence-update-page \
  --page-id '211539684' \
  --title 'Updated title' \
  --content 'h1. Updated body' \
  --content-format 'wiki' \
  --output json
```

## Wrapper behavior

### `jira-mcp`

- Accepts the same generated subcommands as `mcp-atlassian`
- Supports wrapper-only `--output json`
- Adds lean default fields for these read-heavy commands when `--fields` is omitted:
  - `jira-search`
  - `jira-get-issue`
  - `jira-get-board-issues`

Default lean fields:

```text
summary,status,assignee,priority,issuetype,updated
```

### `confluence-mcp`

- Accepts the same generated subcommands as `mcp-atlassian`
- Supports wrapper-only `--output json`
- Does not rewrite tool arguments

## Environment toggles

```bash
# Show binary stderr/logging
ATLASSIAN_CLI_VERBOSE=1

# Disable lean Jira defaults
ATLASSIAN_CLI_LEAN=0

# Override lean Jira fields
ATLASSIAN_JIRA_LEAN_FIELDS="summary,status,assignee"

# Keep pretty JSON instead of compact one-line JSON
ATLASSIAN_CLI_COMPACT_JSON=0
```

## Notes

- `config/mcporter.json` and `scripts/mcp-atlassian` are treated as local generated artifacts.
- The compiled binary inherits auth and URLs from `.mcp.json` via imported config.
- Do not commit generated binaries or copied config if they contain tokens.
